Description and Motivation

CanvasTracker is a web-based academic planning tool designed to reduce the friction students experience when using the standard Canvas interface. Instead of forcing users to click through multiple pages for assignments, quizzes, and announcements, CanvasTracker consolidates everything into a single dashboard with multiple views. The goal is to help students feel more in control of their workload, minimize missed deadlines, and streamline daily planning.

In the default Canvas experience, critical information is scattered across courses, assignment pages, calendars, and announcement feeds. Students must constantly jump between menus to piece together what is due and when. This fragmentation increases cognitive load and makes it easy to overlook tasks, especially when juggling multiple courses and deadlines. Our group set out to explore how an integrated Canvas-aware planner could improve usability, efficiency, and perceived control compared to existing Canvas tools and third-party apps.

Goals and Design Requirements

The main goal was to design and evaluate an integrated Canvas dashboard that consolidates all assignments and deadlines into one interface with multiple views. The interface needed to be quick to scan, simple to interact with, and flexible enough to support different planning styles.

At a system level, CanvasTracker was designed to provide a centralized, low-navigation view of all upcoming assignments and deadlines, reliably synchronize with Canvas so newly released tasks appear without manual copying, allow users to add personal tasks or reminders that sit alongside Canvas items, visually indicate urgency and status with color-coding and progress-like cues, and provide direct hyperlinks from each task back to its original Canvas page.

Key Features

The interface is built around a unified dashboard that can be viewed either as a calendar or as a list. In both modes, tasks are easy to find, filter, and manage without navigating away from the main page. Users can add tasks, set due dates, enable reminders, and mark items as complete.

Color and layout are used to communicate urgency and category. Tasks are grouped by due date, and the calendar view supports expanding a specific day to see all details at once. Direct links jump users into the corresponding Canvas assignment page when they are ready to submit or view more details, reducing the number of clicks required to get from planning to action.

Technology Stack

The project uses a lightweight but flexible stack centered around web standards and the Canvas LMS API. The front end is implemented with HTML5, CSS3, and vanilla JavaScript to keep interactions responsive and easy to maintain. Task lists and calendar views are rendered in the browser, emphasizing clarity and responsiveness.

CanvasTracker communicates with Canvas through a custom API proxy built with a Cloudflare Worker. This layer handles authentication and API requests while shielding client-side code from exposing sensitive tokens. Python scripts were used during development for data analysis and preprocessing tasks related to user feedback and app review text mining.

Online Study: App Review Analysis

During the early research phase, thousands of user reviews from various academic planning and scheduling apps were collected and analyzed. Word clouds helped reveal the most frequently used terms, highlighting recurring concerns and feature requests from real users. These insights guided the design priorities of CanvasTracker and grounded the project in real-world expectations for scheduling tools.

Prototyping Phase

The prototyping phase moved from hand-drawn sketches to interactive digital interfaces. Iteration 1 focused on quick wireframes to explore layout ideas, Iteration 2 translated those sketches into working low- and mid-fidelity prototypes, and Iteration 3 refined the interface into the final high-fidelity design used in the user studies.

Each iteration was evaluated with users, and their feedback shaped how tasks were displayed, how synchronization behaved, and how much control students felt over their schedule.

The first iteration focused on sketching out basic layouts for the task manager and calendar dashboard. Iteration 3: Final Version

The final iteration of CanvasTracker represents a fully refined interface based on insights gathered from user testing, workload assessments, and structured evaluation of earlier prototypes. This version features a polished dark-mode layout, clearer task organization, improved spacing, and streamlined interactions for syncing Canvas assignments and filtering upcoming tasks.

By incorporating consistent feedback across multiple rounds of usability testing, the design balances readability, efficiency, and visual clarity, delivering a clean, modern task management environment that aligns with students’ real needs.

User Study 1: Interface Evaluation Results

After creating early interactive prototypes for both the Calendar View and the List View, users were asked to evaluate each layout based on clarity, usability, and overall intuitiveness. Participants rated statements on a five-point Likert scale ranging from Strongly Disagree to Strongly Agree.

The study provided insight into which layout communicated information more effectively and which interaction patterns felt easiest for users.

Evaluation of the Calendar View. Users generally found the interface visually clear, though some felt the layout was less intuitive than the List View.

Evaluation of the List View. Participants reported stronger usability scores overall, especially for clarity, ease of identifying assignments, and task entry.

User Study 2: Workload and Usability Evaluation

To better understand how users experienced CanvasTracker during hands-on testing, participants completed a NASA Task Load Index assessment and a short usability questionnaire. These evaluations measured perceived mental demand, effort, frustration, and overall interaction satisfaction.

NASA TLX results indicate low mental demand, low effort, and very low frustration when using the system, suggesting that the planner was intuitive and low-stress to interact with.

Usability feedback shows high agreement that the system was easy to use, well organized, and made it easy to find assignments. Users also reported feeling in control and expressed interest in regular use if integrated with Canvas.

Iterations and Overall Findings

Across all iterations, users consistently described the interface as clean and simple, with task creation and the calendar day-expansion feature standing out as particularly useful. Feedback guided improvements such as clearer controls for deleting, editing, or reordering tasks, avoiding duplicates when syncing, showing more tasks per calendar view, and adding options for hiding completed or distant tasks. Suggestions such as class-based color coding and optional time fields for deadlines informed future design directions.

Usability metrics and NASA-TLX workload scores indicated low mental demand and low frustration. Participants reported that CanvasTracker felt intuitive, well organized, and effective for managing a busy schedule. Many preferred it over the native Canvas interface for daily assignment management and filtering, while still relying on Canvas for announcements.

Outcome and Reflection

The final prototypes demonstrate that an integrated, Canvas-aware planner can meaningfully reduce the friction of staying on top of coursework. By simplifying navigation, presenting all tasks in one place, and supporting both Canvas and personal items, CanvasTracker helps students feel more organized and in control of their time.

Future directions include deeper calendar integration, richer color-coding and tagging options, improved task history views, and additional accessibility refinements. With native Canvas integration, CanvasTracker or a similar tool could serve as a primary daily planning surface for students.
